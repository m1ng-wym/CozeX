# Coze 低代码工作流执行计划

## 1. 工作流概述

- **名称**: cn_to_en_translation
- **描述**: 接收用户输入的中文文本，自动翻译为英文输出
- **类型**: 工作流（Workflow）
- **目的**: 实现中文到英文的自动翻译功能，支持日常短句、含数字专有名词的句子、以及较长段落

## 2. 节点架构

| 节点 # | 类型 | 名称 | 用途 |
|--------|------|------|------|
| 1 | 开始 | 开始 | 定义输入参数：input（String类型，中文文本） |
| 2 | 大模型 | 翻译 | 调用大语言模型进行中译英 |
| 3 | 结束 | 结束 | 返回翻译后的英文文本（默认存在，无需添加） |

## 3. 连接顺序

| 步骤 | 从节点 | 从端口 | 到节点 | 到端口 |
|------|--------|--------|--------|--------|
| 1 | 开始 | output（右侧） | 翻译 | input（左侧） |
| 2 | 翻译 | output（右侧） | 结束 | input（左侧） |

## 4. 变量配置

| 节点 | 变量名 | 类型 | 来源/值 |
|------|--------|------|---------|
| 开始 | input | String | 用户输入的中文文本 |
| 大模型 | input | String | {{开始.output}} |
| 大模型 | prompt | String | 请将下面的中文文本翻译成英文，只输出翻译结果：{{input}} |
| 大模型 | model | String | Doubao-pro（或其他合适模型） |
| 大模型 | output_format | String | text |
| 结束 | output | String | {{翻译.output}} |

## 5. 执行顺序

### 步骤 1：创建工作流
1. 导航至 https://code.coze.cn
2. 在页面顶部选择目标工作空间
3. 在左侧导航栏单击**资源库**
4. 在资源库页面右上角，选择 **+资源 > 工作流**
5. 输入工作流名称：`cn_to_en_translation`
6. 输入描述：`接收中文文本，自动翻译为英文`
7. 点击确认创建
8. 页面会自动跳转至工作流编辑页面

### 步骤 2：添加节点
1. 从底部面板拖拽**大模型节点**到画布
2. **注意**：结束节点是默认存在的，无需添加

### 步骤 3：连接节点
1. 点击"优化布局"按钮，等待布局完成（约1.5秒）
2. 获取所有端口的精确坐标
3. 连接开始节点 output → 大模型节点 input
4. 连接大模型节点 output → 结束节点 input
5. 再次点击"优化布局"整理连接线

### 步骤 4：配置节点变量

#### 开始节点配置
- 确认默认输入参数 `input`（String类型）已存在
- 点击展开图标（如有），可添加参数描述：需要翻译的中文文本

#### 大模型节点配置
1. 点击大模型节点
2. 选择模型：`Doubao-pro` 或其他可用模型
3. 配置输入：点击输入框右侧六边形按钮，选择 `开始 - output`
4. 编写用户提示词：
   ```
   请将下面的中文文本翻译成英文，只输出翻译结果，不要任何解释：
   {{input}}
   ```
5. 系统提示词可留空或填写：`你是一个专业的翻译助手`
6. 输出格式选择：`文本`

#### 结束节点配置
1. 点击结束节点
2. 选择返回模式：`返回文本`
3. 在回答内容中引用：`{{翻译.output}}`

### 步骤 5：测试工作流

#### 第一轮测试
- **测试输入**: `你好，很高兴认识你`
- **期望输出**: `Hello, nice to meet you.`
- **成功标准**: 翻译准确，语句通顺

#### 第二轮测试
- **测试输入**: `我今年25岁，来自北京`
- **期望输出**: `I am 25 years old and come from Beijing.` 或 `I'm 25 years old and from Beijing.`
- **成功标准**: 数字25正确，北京翻译准确

#### 第三轮测试
- **测试输入**: `今天天气非常好，阳光明媚，我计划和家人一起去公园野餐。我们会带上水果、三明治和饮料。这是一个放松身心的好机会。`
- **期望输出**: 英文段落，意思准确传达
- **成功标准**: 较长文本翻译流畅，无语法错误

#### 测试失败处理
- 如果测试出现错误，检查节点连接和变量配置
- 确保变量引用通过六边形按钮选择（不是直接输入文本）
- 检查大模型提示词是否清晰
- 查看错误信息提示，定位问题节点
- 修复后重新测试，直到三轮全部通过

### 步骤 6：发布工作流
- **版本**: v0.0.1
- **描述**: 初始版本，实现中文到英文的基本翻译功能
- **发布条件**: 三轮测试均准确无误

## 7. 特殊注意事项

1. **变量引用必须通过六边形按钮选择**，不能直接输入文本
2. **连接必须通过鼠标拖拽**，不能简单点击，50步每步40ms
3. **每步操作后等待响应**，避免操作过快导致失败
4. **连接前必须点击"优化布局"**，确保端口坐标准确
5. **翻译提示词要明确**：要求"只输出翻译结果"，避免大模型添加额外解释
6. **测试时逐轮验证**：每一轮测试通过后再进行下一轮
7. **所有测试必须成功**：任何一轮测试失败都需要排查解决后才能继续
