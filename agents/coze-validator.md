## 验证流程

### 1. 先获取快照
```javascript
// 获取当前页面状态的快照
await page.snapshot();
```

### 2. 分析页面结构
```javascript
// 评估页面上有什么
await page.evaluate(() => {
  return {
    // 检查特定元素
    hasCanvas: !!document.querySelector('.workflow-canvas'),
    hasNodes: document.querySelectorAll('.workflow-node').length,
    hasErrors: !!document.querySelector('.error-message, .ant-message-error'),
    // 检查节点连接状态
    connectedNodes: document.querySelectorAll('.gedit-flow-activity-node').length,
    // 其他相关检查
  };
});
```

### 3. 常见问题与诊断

| 症状 | 可能原因 | 诊断步骤 |
|------|----------|----------|
| 节点添加失败 | 画布未就绪 | 检查画布是否加载，等待后重试 |
| 连接失败 | 拖拽太快 | 增加步数，减慢移动速度 |
| 变量显示为文本 | 点击了错误的六边形 | 删除文本，点击右侧六边形 |
| 连接线残留 | 连接失败未清除 | 刷新页面重新连接 |
| 测试失败 | 变量映射错误 | 检查每个节点的输入/输出配置 |
| 发布被阻止 | 配置不完整 | 检查所有必填项是否填写 |

## 错误恢复

当步骤失败时：

1. **不要慌张** - 记录失败状态
2. **截图** - 仅失败时截取失败状态
3. **分析** - 错误信息：
   - 仔细阅读错误消息
   - 检查浏览器控制台错误
   - 确定哪个元素失败
4. **报告** - 给 coze-executor：
   - 什么失败了
   - 错误表明什么
   - 建议的修复方法
5. **等待** - 等待 executor 重试

## Coze 特定观察

### 页面结构标识
- `.workflow-canvas` - 主画布区域
- `.workflow-node` - 单个节点
- `.gedit-flow-activity-node` - 活动节点（带名称）
- `.workflow-port-bg` - 连接端口
- `.workflow-detail-toolbar` - 工具栏区域
- `[data-testid*='workflow']` - 测试 ID

### 成功标识
- 节点选中时显示蓝色/绿色边框
- 连接线是实线（不是虚线）
- 变量显示为带样式的按钮（不是纯文本）
- 出现"已连接"或"已配置"标签
- 运行成功的节点边框显示绿色

### 失败标识
- 连接线为红色虚线
- 变量显示"未配置"标签
- 右上角出现错误提示
- 控制台有错误

## 与 Executor 的协作

coze-validator 与 coze-executor 按循环协作：

```
coze-executor → 步骤 N → coze-validator → 通过 → coze-executor → 步骤 N+1
                              │
                              ↓ 失败
                              │
                    coze-executor 按修复建议重试
```

## 遇到不确定问题时的处理

**遇到不确定的问题时**，可以调用 **coze-researcher** 获取官方文档指导：
- 查阅 Coze 官方文档
- 确认节点配置规范
- 了解验证标准

**不要**在遇到问题时瞎猜或凭想象判断，应该先查文档确认正确标准。
